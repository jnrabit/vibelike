### 5. Risiken

**Architektur-Risiken:**
1. **151 KB Single-File** — `workflow_agent.py` ist ein Monolith-in-einem-Monolith. Jede neue Feature-Ergänzung erhöht die Komplexität einer bereits schwer navigierbaren Datei. Regression-Guard schützt vor Größen-Kollaps, aber nicht vor schleichender Komplexitätszunahme.

2. **Iteration-Loop ohne Konvergenz-Garantie** — `max_iterations` begrenzt Loops, aber die Failure-Analysis → Phase-1-Rücksprung kann bei systematischen Fehlern (z.B. fehlendes Paket) endlos dieselbe falsche Korrektur generieren.

3. **Council-Modus: Synthese-Bottleneck** — Wenn `council_claude` und `council_gemini` parallel generieren, aber `council_synth` (ebenfalls Claude Sonnet) synchron synthetisiert, ist der Parallelismus nur auf der Generierungs-Ebene, nicht auf der Entscheidungs-Ebene.

4. **Healthpoint warn-only** — `check_drift` ist warn-only. Phasen-Drift wird geloggt aber nicht gestoppt. Bei einem schleichenden Ziel-Drift über 3 Iterationen akkumuliert sich der Fehler unbemerkt.

5. **`_build_existing_files_context` als Regression-Prävention** — das ist ein Symptom-Fix (Kontext-Injection gegen Mini-Stubs), nicht eine strukturelle Lösung. Wenn der Kontext zu groß wird (viele große Dateien), wird der Prompt token-kritisch.

### 6. Was ist DISTINKT an diesem Projekt?

**Pattern 1: Authoritative-File-List-Sandwich als Halluzinations-Guard**

In anderen Codebases gibt es Prompt-Engineering, aber selten ein dediziertes Methoden-Konstrukt das die Dateiliste als "verbindliche Quelle der Wahrheit" in jeden Prompt injiziert:

```python
def _authoritative_file_list(self) -> str:
    """Liste ALLER .py-Dateien im Projekt-Root — als verbindliche Quelle der Wahrheit."""
    files = sorted(p.name for p in self.root.glob("*.py"))
    return "\n".join(f"  - {f}" for f in files)
```

Das ist ein **struktureller Anti-Halluzinations-Mechanismus** — nicht als Kommentar oder Konvention, sondern als Methode die bei jedem Prompt-Build aufgerufen wird. Das "Sandwich" (Liste am Anfang UND Ende des Prompts) ist eine bekannte Technik, aber die Implementierung als Methode mit `glob("*.py")` (live, nicht hardkodiert) ist ungewöhnlich sauber.

**Pattern 2: Regression-Guard als Pre-Write-AST-Diff**

```python
# _check_regression (vor dem Write): fängt destruktive Überschreibungen
# AST-Symbol-Verlust + Größen-Kollaps
```

Der Befund `650→26 Zeilen` (aus `_build_existing_files_context`-Docstring) zeigt, dass das System bereits echte Selbstzerstörung erlebt hat. Die Antwort war kein Prompt-Fix, sondern ein **deterministischer Pre-Write-Guard** der AST-Symbole zählt und Größen-Kollaps stoppt. Das ist in LLM-Coding-Agents selten — die meisten vertrauen auf den LLM-Output ohne strukturelle Absicherung.
